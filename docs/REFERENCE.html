Subsets

Generate the power set via backtracking — the gateway pattern for combinatorial search

MEDIUM
LC 78

Problem Statement

Challenge

Given an array nums of unique integers, return all possible subsets (the power set).

  • Input: array of unique integers.
  • Output: list of all subsets, including the empty subset.
  • No duplicate subsets allowed; order of subsets in the output does not matter.

Example 1

Input / Output
Input : nums = [1, 2, 3] Output: [[], [1], [2], [1,2], [3], [1,3], [2,3], [1,2,3]] # Every combination of including / excluding each of # the 3 elements produces 2³ = 8 subsets.

Example 2

Input / Output
Input : nums = [7] Output: [[], [7]] # The only element can be included or excluded, # giving exactly 2 subsets.
💡
Key Insight: At each index there are exactly two choices — include the element or skip it. Traversing all such binary decisions from left to right produces every possible subset exactly once. For an array of length n this yields 2ⁿ subsets, and copying each subset of average length n/2 gives the O(n · 2ⁿ) time bound.

Core Concepts

Pattern Definition

Backtracking is depth-first search over a decision tree where each node represents a partial solution. At each step we make a choice, recurse, then undo that choice (backtrack) before trying the alternative.

Here, the decision at every index is include / skip. There is no pruning needed because every path to a leaf is valid — every leaf is a complete subset.

Core Loop

  1. Make a choice (include or skip the element)
  2. Recurse with the updated state
  3. Undo the choice (subset.pop()) — backtrack
  4. Try the alternative
⚠️
Important Caveat: Always append a copy of subset, not the list itself. res.append(subset) stores a reference — backtracking will mutate that reference later, corrupting previously recorded subsets. Use res.append(subset.copy()) or res.append(subset[:]).

Decision Process / State View

State at each recursive call: (i, subset) — current index and elements chosen so far.

Decision Tree — nums = [1, 2, 3]
dfs(i=0, subset=[]) ├── include nums[0] → dfs(i=1, subset=[1]) │ ├── include nums[1] → dfs(i=2, subset=[1,2]) │ │ ├── include nums[2] → dfs(i=3) → record [1,2,3] │ │ └── skip nums[2] → dfs(i=3) → record [1,2] │ └── skip nums[1] → dfs(i=2, subset=[1]) │ ├── include nums[2] → dfs(i=3) → record [1,3] │ └── skip nums[2] → dfs(i=3) → record [1] └── skip nums[0] → dfs(i=1, subset=[]) ├── include nums[1] → dfs(i=2, subset=[2]) │ ├── include nums[2] → dfs(i=3) → record [2,3] │ └── skip nums[2] → dfs(i=3) → record [2] └── skip nums[1] → dfs(i=2, subset=[]) ├── include nums[2] → dfs(i=3) → record [3] └── skip nums[2] → dfs(i=3) → record []
📊
Total tree nodes: 2ⁿ⁺¹ − 1. Leaf nodes (depth = n): exactly 2ⁿ, one per subset.

Complete Solution (Primary)

Python
from typing import List class Solution: def subsets(self, nums: List[int]) -> List[List[int]]: res = [] subset = [] def dfs(i): if i >= len(nums): res.append(subset.copy()) return subset.append(nums[i]) dfs(i + 1) subset.pop() dfs(i + 1) dfs(0) return res

Knowledge Check

Q1 — Complexity: What is the time complexity of the backtracking solution?

A) O(n²)
B) O(2ⁿ)
C) O(n · 2ⁿ)
D) O(n log n)
Answer: C — There are 2ⁿ leaf nodes (subsets). At each leaf we copy subset, which takes O(n) worst case. Multiplying gives O(n · 2ⁿ).

Q2 — Correctness Invariant: Which invariant guarantees no duplicate subsets are generated?

A) We sort nums before recursing.
B) We always recurse left-to-right (i → i+1), never revisiting earlier indices.
C) We use a set to de-duplicate results.
D) We prune branches when the current element has been seen before.
Answer: B — Because we only advance the index forward (dfs(i+1)), each element is decided at most once per path, preventing duplicates.

Q3 — Edge Case: What does the algorithm return for nums = []?

A) None
B) []
C) [[]]
D) Raises an IndexError
Answer: C — dfs(0) is called and immediately hits the base case i >= len(nums) (0 ≥ 0), recording a copy of the empty subset — giving [[]].

Q4 — Implementation Pitfall: Why is res.append(subset) wrong in the base case?

A) subset is a tuple and cannot be appended to a list.
B) It stores a reference; later pop() calls mutate the stored object.
C) It skips the empty subset.
D) It causes infinite recursion.
Answer: B — subset is a mutable list. Appending it directly stores a reference. Every subsequent subset.pop() during backtracking modifies the same object already in res, corrupting results. Use subset.copy().

Alternative Approaches

Alternative 1: Iterative Building

Intuition: Start with [[]]. For each number, duplicate every existing subset and append the number to each duplicate. This doubles the list with each element.

Python
from typing import List class Solution: def subsets(self, nums: List[int]) -> List[List[int]]: res = [[]] for num in nums: res += [subset + [num] for subset in res] return res
  • Time: O(n · 2ⁿ)
  • Space: O(n) extra; O(2ⁿ) for output.
  • When to prefer: Avoids recursion overhead; useful when stack depth is a concern.

Alternative 2: Bit Manipulation

Intuition: There are 2ⁿ integers from 0 to 2ⁿ−1. Each bit j of integer i encodes whether nums[j] is included in the subset for mask i.

Python
from typing import List class Solution: def subsets(self, nums: List[int]) -> List[List[int]]: n = len(nums) res = [] for i in range(1 << n): subset = [nums[j] for j in range(n) if (i & (1 << j))] res.append(subset) return res
  • Time: O(n · 2ⁿ)
  • Space: O(n) extra; O(2ⁿ) for output.
  • When to prefer: Elegant one-liner logic; shows binary intuition. Note: limited to ~30 elements in C/Java due to integer overflow.

Common Mistakes

Mistake 1: Appending a reference instead of a copy

Wrong
if i >= len(nums): res.append(subset) # stores reference return
Correct
if i >= len(nums): res.append(subset.copy()) # snapshot return

Why it fails: subset is mutated by every pop() during backtracking. All entries in res that point to the same object get corrupted.

Quick check: After the call, print each element of res and verify they are distinct objects with id().

Mistake 2: Forgetting the empty subset in the iterative approach

Wrong
res = [] # missing the seed for num in nums: res += [subset + [num] for subset in res]
Correct
res = [[]] # seed with empty subset for num in nums: res += [subset + [num] for subset in res]

Why it fails: Without [[]] as the seed, the first iteration produces nothing and all subsequent iterations remain empty.

Quick check: Confirm [] appears in the final output.

Mistake 3: Re-using the current index instead of advancing

Wrong
def dfs(i): ... for j in range(i, len(nums)): subset.append(nums[j]) dfs(j) # passes j, not j+1 subset.pop()
Correct
def dfs(i): ... subset.append(nums[i]) dfs(i + 1) # advance by 1 subset.pop() dfs(i + 1)

Why it fails: Passing j instead of j+1 allows the same index to be picked again, generating duplicates like [1,1].

Quick check: Verify no subset has repeated elements when the input has no duplicates.

Mistake 4: Integer overflow in bitmask for large n

Wrong (C/Java)
// 1 << 31 overflows 32-bit int for (int i = 0; i < (1 << n); i++)
Correct (C/Java)
// use long or 1LL when n > 30 for (long i = 0; i < (1L << n); i++)

Why it fails: In statically-typed languages 1 << 31 overflows a 32-bit int. Python is exempt (arbitrary-precision ints), but the mistake surfaces when translating code.

Quick check: Confirm n satisfies problem constraints (here n ≤ 10, so no risk in Python).

What is the base case for the backtracking solution to Subsets?

(click to flip)

When i >= len(nums) — we have made a decision for every element, so we append a copy of subset to res and return.

Why is the time complexity O(n · 2ⁿ) and not just O(2ⁿ)?

(click to flip)

There are 2ⁿ subsets (leaves in the decision tree). Recording each one requires copying subset, which takes O(n) time worst case. The copy cost gives the extra factor of n.

What is the core backtracking pattern for subset generation?

(click to flip)

Include the element → recurse → pop (backtrack) → recurse again without the element. Two recursive calls per index, advancing i by 1 each time, covering all 2ⁿ combinations.
